1 # OMPExceptionCatcher
2 A lightweight class for managing C++ exception handling strategies in OpenMP code.
3 
4 ## Motivation
5 OpenMP code must catch any exceptions that may have been thrown before exiting the OpenMP block.
6 This class acts as lightweight wrapper that allows an arbitrary function or lambda expression to be run
7 safely and efficiently in OMP even if it might throw exceptions. We employ one of 4 possible strategies
8 as determined By the OMPExceptionCatcher::Strategies enum.
9 
10  ## Excepton Catching Strategy's
11  * `OMPExceptionCatcher::Strategies::DoNotTry` -- Don't even try, this is a null op to completely disable
12  this class's effect.
13  * `OMPExceptionCatcher::Strategies::Continue` -- Catch exceptions and keep going
14  * `OMPExceptionCatcher::Strategies::Abort` -- Catch exceptions and abort
15  * `OMPExceptionCatcher::Strategies::RethrowFirst` -- Re-throws first exception thrown by any thread

19  ## Including OMPExceptionCatcher in your OpenMP project
20 Since OMPExceptionCatcher is header-only, he easiest way to use it is via the [git subrepo](https://github.com/ingydotnet/git-subrepo) plugin. Unlike the traditional `git submodule` command, `git subrepo` is transparent to other users of your repository, and solves many of the irksome issues prevalent with the submodule approach. Follow the [git subrepo install guide](https://github.com/ingydotnet/git-subrepo#installation-instructions) to install on your development machine.
