5 Currently building has only been tested on linux hosts.

7 Mappel has been tested with modern GCC-7.2.0 as well as earlier GCC's to 4.9.4 (which is still required for Matlab integration). In order
8 to maintain compatibility with gcc-4.9.4 and still use modern C++14 features we use the `-std=c++1y` GCC standard when compiling.

10 ## Dependencies
11 
12 Several standard numerical packages are required to build Mappel. Most distributions should have development versions of these packages which provide the include files and
13 other necessary development files for the packages.
14 
15 * [Armadillo](http://arma.sourceforge.net/)
16 * [Boost](http://www.boost.org/)
17 * [TRNG](https://github.com/rabauke/trng4)
18 * BLAS
19  * Requires support for 64-bit integers.
20  * [Netlib BLAS Reference](http://www.netlib.org/blas/)
21 * LAPACK
22  * Requires support for 64-bit integers.
23  * [Netlib LAPACK Reference](http://www.netlib.org/lapack/)

48 ### External Projects
49 
50 Mappel also depends on several small Github projects which for now are maintained in separate repositories.
51 
52 - [BacktraceException](https://github.com/markjolah/BacktraceException) - A library to provide debugging output
53  on exception calls. Important for Matlab debugging.
54 - [ParallelRngManager](https://github.com/markjolah/ParallelRngManager) - A simple manager for easily deploying a set of RNG
55  parallelized over a set number of threads, using the TRNG parallel RNG library.
56 - [PriorHessian](https://github.com/markjolah/ParallelRngManager) - The PriorHessian library allows fast
57  computation of log-likelihood and derivatives for composite priors.
58 
59 If these libraries do not exist on the build system or at `CMAKE_INSTALL_PREFIX`, they are
60 automatically downloaded, configured and installed as external dependencies during the CMake configure phase.
61 
62 Normally the Github current versions of the external dependencies are used. To use the HEAD revision of a local git repository,
63 the following Environment variables can be set:
64 
65 - `BacktraceExceptionURL` - Local directory or git URL for the BacktraceException library [optional] Default to use the HEAD version from Github
66 - `ParallelRngManagerURL` - Local directory or git URL for the ParallelRngManager library
67 - `PriorHessianURL` - Local directory or git URL for the PriorHessian library

71 ### Linux

74 git clone https://github.com/markjolah/Mappel.git
75 cd Mappel
76 ./build.sh
77 ```

80 ## Debugging
81 
82 CMake variable `CMAKE_BUILD_TYPE=Debug` will configure the debug build and all libraries and executables will have a `.debug`
83 suffix.
84 
85 A convenience script exists to only build the debug versions of the libraries in the local `_build` and `_install` directories
86 
87 ```
88 ./build.debug.sh
89 ```
90 Tips:
91 * Try running `VERBOSE=1 make` inside the `_build/Debug` directory to debug the build/link command lines generated by CMAKE.

93 ## Python support
94 Matlab support is enabled by setting the CMake option `-DOPT_PYTHON=1`. At the moment only python 3 is supported.
95 
96 Several CMake variable can control for which python version modules are built.
97  * `MAPPEL_PYTHON_VERSIONS` - List of python X.Y versions seperated by ";" to build modules for (e.g., "3.4;3.5;3.6")
98  * `MAPPEL_PYTHON_EXECUTABLE` - Name or full path to python executable on the system for which to build (e.g., python3).
99 
100 Mappel uses [pybind11](https://pybind11.readthedocs.io/en/stable/) to compile modules for each Mappel Model
101 class.
102 
103 ### Python development workflow
104 
105 The Mappel python package environment is created in the build tree at `build_dir/python`, as part of the CMake build process.
106 There is a standard [setuptools](https://setuptools.readthedocs.io/en/latest/setuptools.html#) `setup.py` that can be used to build binary distributions and also to install to the local system.
107 
108 The CMake install process will automatically install the python .egg using `setup.py` under the `CMAKE_INSTALL_PREFIX` directory.
109 
110 In order to be able to develop the code at the root `mappel/python/` git repository while running and testing the mappel package without having to `make install` on every small change to python code, we use the [developer mode](https://setuptools.readthedocs.io/en/latest/setuptools.html#development-mode) install option provided by setuptoools. In fact, we have made it even easier to use, by making an alias `localdevelop`
111 
112 ```
113 $ cd _build/Debug/python
114 $ python setup.py localdevelop
115 $ python -m mappel
116 ```

118 ## Matlab support
119 
120 
121 Matlab support is enabled by setting the CMake option `MATLAB=on`. This brings in an additional external dependency,
122 * [MexIFace](https://github.com/markjolah/MexIFace) - A cross-platform Matlab/C++ class-based interface wrapper for generating .mex files.
123 
124 The following environment variables control the Matlab build process
125 * `MexIFaceURL`: Local directory or git URL for the MexIface library (Matlab Support). [optional] Default to use the `HEAD` version from Github
126 * `MATLAB_LIBS_ROOT`: [Optional] Local path to find Matlab core shared libraries to link against (overrides default search paths).
127  Must contain subdirectory structure `($MATLAB_ARCH)/$(MATLAB_VERSION)/{bin,extern}`. `MATLAB_ARCH` is [glnxa64, maci64, win64].
128 * `MATLAB_ROOT_GLNXA64`: Necessary for Matlab. Location of the Matlab `glnxa64` version to link against.
129 
130 ## Cross-building to Win64
131 
132 The following Environment variables control the Win64 cross-build environment necessary to compile win64 binaries
133 * `MXE_ROOT` Local directory root of the MXE Win64 cross environment. Necessary
134  For Win64 cross-compiling only.
135 
136 * `MATLAB_ROOT_WIN64` Necessary for Matlab on Win64 cross build. Location of the Matlab `win64` version to link against.
137 
138 ## Cross-building to OSX
139 
140 * `OSXCROSS_ROOT`: Local directory root of the OSXCross OSX 64-bit cross environment.
141  Necessary for OSX cross-compiling only.
142 
143 * `MATLAB_ROOT_MACI64`: Necessary for Matlab on OSX cross build. Location of the Matlab `maci64` version to link against.
